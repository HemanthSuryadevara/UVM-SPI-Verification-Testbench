## 📌 Project Overview

This project implements a complete **UVM (Universal Verification Methodology)**
testbench for an SPI (Serial Peripheral Interface) Master designed from scratch
in SystemVerilog. The goal is to verify the functional correctness of the
`spi_master` RTL module using industry-standard UVM verification techniques.

This is a **beginner-level learning project** — every file is heavily commented
to explain not just what the code does, but **why** each component exists and
how it fits into the overall UVM architecture.

## 🔬 Protocol Overview — SPI

SPI (Serial Peripheral Interface) is a 4-wire synchronous serial communication protocol:

| Signal | Direction      | Description                        |
|--------|----------------|------------------------------------|
| SCLK   | Master → Slave | Serial clock generated by master   |
| MOSI   | Master → Slave | Master Out Slave In — data output  |
| MISO   | Slave → Master | Master In Slave Out — data input   |
| CS_N   | Master → Slave | Chip Select — active LOW           |

**SPI Master FSM:**
```
IDLE → CS_SETUP → TRANSFER → CS_HOLD → IDLE
```

## 📁 Project Structure

```
spi/
├── rtl/
│   ├── spi_master.sv           ← SPI Master RTL (4-state FSM)
│   └── spi_slave.sv            ← Simple SPI Slave model
│
└── tb/
    ├── spi_if.sv               ← SystemVerilog Interface
    ├── spi_seq_item.sv         ← Transaction (sequence item)
    ├── spi_sequencer.sv        ← Sequencer (queue manager)
    ├── spi_driver.sv           ← Driver (signal wiggler)
    ├── spi_monitor.sv          ← Monitor (passive bus observer)
    ├── spi_scoreboard.sv       ← Scoreboard (checker)
    ├── spi_agent.sv            ← Agent (driver+monitor+sequencer)
    ├── spi_env.sv              ← Environment (agent+scoreboard)
    ├── spi_sequence.sv         ← Sequences (test stimulus)
    ├── spi_test.sv             ← Test classes
    ├── spi_pkg.sv              ← Package (bundles all TB files)
    └── tb_top.sv               ← Top module (DUT + IF + clock)
```

## 🧱 UVM Architecture

```
tb_top  (module — hardware world)
    │
    ├── spi_if        (interface — signal bundle)
    ├── spi_master    (DUT)
    └── run_test()    (hands over to UVM)
            │
            ▼
    spi_basic_test    (uvm_test)
            │
            ▼
        spi_env       (uvm_env)
            │
            ├── spi_agent
            │       ├── spi_sequencer
            │       ├── spi_driver      ──► drives DUT
            │       └── spi_monitor     ──► watches bus
            │               │ ap.write()
            └── spi_scoreboard
                    └── write() checks TX == RX
```

## 🚀 How to Run

### Set Environment
```bash
export XCELIUM_HOME=/opt/cadence/installs/XCELIUM2503/tools.lnx86
export UVM_HOME=$XCELIUM_HOME/methodology/UVM/CDNS-1.2/sv
export PATH=$XCELIUM_HOME/bin:$PATH
```

### Run Simulation
```bash
cd tb/

xrun -sv \
    +incdir+$UVM_HOME/src \
    $UVM_HOME/src/uvm_pkg.sv \
    ../rtl/spi_master.sv \
    spi_if.sv \
    spi_pkg.sv \
    tb_top.sv \
    -top tb_top \
    +UVM_TESTNAME=spi_basic_test \
    +UVM_VERBOSITY=UVM_MEDIUM \
    -timescale 1ns/1ps \
    -access +rwc
```

### Expected Output
```
UVM_INFO [RNTST] Running test spi_basic_test...
UVM_INFO [SPI_SBD] PASS ✓  TX=0xA5 matched RX=0xA5
  SPI SCOREBOARD SUMMARY
  PASS : 10  FAIL : 0
UVM_ERROR :   0
UVM_FATAL :   0
```

## ✅ What Gets Verified

- SPI master FSM transitions
- CS_N asserts correctly at start and end of transfer
- SCLK generates at correct frequency
- Data shifts out MSB first on MOSI
- Loopback integrity: tx_data === rx_data
- DUT busy and done signals

## 📚 Key UVM Concepts Demonstrated

- `uvm_sequence_item` with rand fields and constraints
- `uvm_driver` with virtual interface and clocking blocks
- `uvm_monitor` with analysis port
- `uvm_scoreboard` with analysis imp
- `uvm_config_db` set/get for interface passing
- `raise_objection` / `drop_objection`
- UVM factory registration
- UVM phases: build → connect → run → report
